# Reasoning Profile

## Purpose
- 整理 Cheap 在不同題材上會重複使用的通用思考模式。
- `Reasoning` 是 Cheap 系統的第一主幹，不是題材分類。

## Core Patterns
- `反例打臉`
  - 拿更強反例拆掉原本說法。
  - 常見材料：對手說法、反例、反問。
- `荒謬成本換算`
  - 把抽象要求換算成金錢、工時、流程成本。
  - 常見材料：數字、時間、人力。
- `誘因白話拆解`
  - 把表面說法翻成白話，指出背後誘因。
  - 常見材料：原話、權責、角色關係。
- `歷史國際映射`
  - 拉外部案例校正本地判斷。
  - 常見材料：歷史事件、國外案例、制度差異。
- `情境推演判斷`
  - 列劇本後判斷最可能版本。
  - 常見材料：角色習性、限制條件、版本分支。
- `風險收益審判`
  - 先檢查一個說法或陰謀是否符合行動者的風險收益。
  - 常見材料：收益、代價、角色地位。
- `抽象議題翻成生活常識`
  - 把制度問題翻成一般人一看就懂的日常畫面。
  - 常見材料：制度機制、生活場景、受影響對象。
- `先定調再追責`
  - 先判決，再往上追責。
  - 常見材料：事件摘要、責任角色、價值判斷。

## Resolution Rules
- 若 `plain_language_signals + incentive language` 很強，優先 `誘因白話拆解`
- 若 `cost_signals` 很強，優先 `荒謬成本換算`
- 若 `history_international_signals` 很強，優先 `歷史國際映射`
- 若 `scenario_signals` 很強，優先 `情境推演判斷`
- 若 `risk_reward_signals` 很強，優先 `風險收益審判`
- 若 `counterexample_signals` 很強，優先 `反例打臉`
- 若事件主要是把制度翻成生活場景，優先 `抽象議題翻成生活常識`
- 若篇幅短且情緒先行，才優先 `先定調再追責`

## Pattern Notes
- `抽象議題翻成生活常識`
  - 常見子套路：標準漂移、制度失能可白話化、體系拆台、公信力受損
  - 收尾不要停在「很扯」，要落到制度信任怎麼被搞壞

## Pattern Rules
- 一篇文只選一個主 `Reasoning`
- 其他 pattern 只能作 supporting move，不可把主線拆散
- 材料不足以支撐主 pattern，就先回 research 補資料
