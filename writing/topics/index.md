# Topic Adaptation

## Role
- topic 是第三層。
- 它不是主腦，不負責先選怎麼想。
- 它的工作是：在 reasoning 與 route 都選好後，補題材材料、語氣與禁忌。

## Topic Rules
- `Topic` 可以多選。
- 當一個事件同時跨政治、治理、設計時，不重選 reasoning，只補多個 topic。

## Topic Types
### `topic/partisan_attack`
- 常見材料：原話、前後立場、時間序列、雙標反例
- 常見禁忌：不要只剩情緒發洩，還是要有話術拆解

### `topic/public_governance`
- 常見材料：政策原文、權責、流程、成本、執行落差
- 常見禁忌：不要只寫立場，不寫制度怎麼運作

### `topic/traffic_governance`
- 常見材料：道路設計、事故場景、責任歸屬、外部制度
- 常見禁忌：不要把結構問題縮成單一駕駛道德問題

### `topic/design_branding`
- 常見材料：品牌沿革、對手說法、成本、辨識度、記憶點
- 常見禁忌：不要只說美醜，要說正當性與公共成本

### `topic/international_security`
- 常見材料：國際案例、歷史前例、角色習性、限制條件
- 常見禁忌：不要只做立場喊話，要有標尺

### `topic/tech_media_ai`
- 常見材料：截圖、查核、輸出對照、平台語境
- 常見禁忌：不要把科技題寫成空泛神話或恐慌

### `topic/lifestyle_consumer`
- 常見材料：價格、體驗差異、生活場景、國外比較
- 常見禁忌：不要只講品味，要講制度或成本

### `topic/history_counterfactual`
- 常見材料：史料、既定印象、反事實條件、現代映射
- 常見禁忌：不要把小說印象直接當正史
