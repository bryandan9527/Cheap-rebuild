# Route Taxonomy

## Role
- route 是第二主幹。
- 它回答的是：這篇要怎麼鋪。
- 同一個 reasoning 可以接不同 route，但單篇只收斂到一條主 route。

## Routes
### `先拋說法再反例拆解`
- 節奏：`Hook -> 爭議說法 -> 反例連發 -> 邏輯反打 -> 判決`
- 適合 reasoning：`反例打臉`

### `先摘要再翻成白話`
- 節奏：`Hook -> 新聞摘要 -> 白話翻譯 -> 誘因拆解 -> 反諷收尾`
- 適合 reasoning：`誘因白話拆解`、`抽象議題翻成生活常識`

### `先算帳再放大荒謬`
- 節奏：`Hook -> 數字 -> 成本換算 -> 荒謬類比 -> 判決`
- 適合 reasoning：`荒謬成本換算`

### `先拉外部案例再映射當下`
- 節奏：`Hook -> 外部案例 -> 對照差異 -> 拉回本案 -> 判決`
- 適合 reasoning：`歷史國際映射`

### `先列劇本再收斂判斷`
- 節奏：`Hook -> 多版本推演 -> 劇本篩選 -> 最可能結果 -> 補刀`
- 適合 reasoning：`情境推演判斷`

### `先定調再往上追責`
- 節奏：`Hook -> 定調 -> 事件責任 -> 上層責任 -> 判決`
- 適合 reasoning：`先定調再追責`

### `先驗證可置信性再下判決`
- 節奏：`Hook -> 指控摘要 -> 風險收益比較 -> 常識檢驗 -> 判決`
- 適合 reasoning：`風險收益審判`

## Topic Adaptation Rule
- route 不直接決定題材。
- 題材適配放在 `topics/` 處理。
- 同一條 route 可以同時掛多個 `Topic`，但節奏不變。
