# Writing System

## Main Flow
`EventBrief -> ReasoningDecision -> RouteDecision -> TopicDecision -> Draft`

## What Each Layer Does
- `EventBrief`
  - 整理事件材料，不做判斷。
- `ReasoningDecision`
  - 決定這件事主體要怎麼想。
- `RouteDecision`
  - 決定這篇文要怎麼鋪。
- `TopicDecision`
  - 補題材材料、語感與禁忌。
- `Draft`
  - 產出 Cheap 貼文草稿。

## Layer Explanations
### Event
- 回答：現在在講什麼事。
- 內容：事件摘要、原話、數字、反例、限制。

### Reasoning
- 回答：這件事最適合怎麼打。
- 它是主腦，不是題材標籤。
- 例：
  - `反例打臉`
  - `誘因白話拆解`
  - `荒謬成本換算`

### Route
- 回答：文章要怎麼鋪。
- 它不是思考模式，而是節奏選擇。
- 例：
  - `先摘要再翻成白話`
  - `先拋說法再反例拆解`
  - `先算帳再放大荒謬`

### Topic
- 回答：這條寫法落到哪些題材時，要補哪些材料。
- 它不是第一關，只是適配層。
- 允許多選。

## Selection Order
1. 先整理 `EventBrief`
2. 單選 `Reasoning`
3. 單選 `Route`
4. 多選 `Topic`
5. 組裝 `Draft`

## Single / Multi Rules
- `Reasoning` 單選
- `Route` 單選
- `Topic` 多選

## Draft Rules
- 不能像新聞摘要
- 必須有明確 angle
- 必須至少有一種 cheap 推進方式：
  - 反例
  - 數字
  - 類比
  - 邏輯反打
  - 風險收益比較
- 收尾必須是判決、補刀或反諷，不是中性總結
