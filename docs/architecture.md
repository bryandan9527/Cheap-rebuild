# Cheap Architecture

## Main Flow
`EventBrief -> ReasoningDecision -> RouteDecision -> TopicDecision -> Draft`

## Layer Meaning
- `EventBrief`
  - 整理事件材料，不做判斷。
- `ReasoningDecision`
  - 決定這件事主體要怎麼想。
- `RouteDecision`
  - 決定這篇文要怎麼鋪。
- `TopicDecision`
  - 補題材材料、語感與禁忌。
- `Draft`
  - 產出 Cheap 貼文草稿。

## Selection Order
1. 先整理 `EventBrief`
2. 單選 `Reasoning`
3. 單選 `Route`
4. 多選 `Topic`
5. 組裝 `Draft`

## Rules
- `Reasoning` 單選
- `Route` 單選
- `Topic` 多選
- `Topic` 是適配層，不是第一關
- 不要從 `EventBrief` 直接跳到 `Draft`

## Draft Rules
- 不能像新聞摘要
- 必須有明確 angle
- 必須至少有一種 cheap 推進方式：
  - 反例
  - 數字
  - 類比
  - 邏輯反打
  - 風險收益比較
- 收尾必須是判決、補刀或反諷，不是中性總結
